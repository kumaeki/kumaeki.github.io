在Mac上编译openjdk9

MAC的OS版本 10.15.6。

安装最新版本的xcode，从applestore下载安装。

安装CommandLineTools，可以在命令行调用xcode。安装11版本。

xcode-select -install

openjdk是用mercurial 管理的，所以先安装 mercurial。

brew install mercurial

然后clone代码。mercurial和git很像，非常像，可以基本当作一个建议版本的git来使用。

hg clone http://hg.openjdk.java.net/jdk/jdk14

检查配置
bash configure

configure: error: No xcodebuild tool and no system framework headers found, use --with-sysroot or --with-sdk-name to provide a path to a valid SDK
/Users/sou/jdk14/build/.configure-support/generated-configure.sh: line 82: 5: Bad file descriptor
configure exiting with result code 1

报错，提示找不到现存的java环境。那么更改一下xocde的build路径。

sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer

然后再试试检查配置

bash configure

configure: Found potential Boot JDK using well-known locations (in /Library/Java/JavaVirtualMachines/jdk1.8.0_221.jdk)
configure: Potential Boot JDK found at /Library/Java/JavaVirtualMachines/jdk1.8.0_221.jdk/Contents/Home is incorrect JDK version (java version "1.8.0_221"); ignoring
configure: (Your Boot JDK version must be one of: 13 14)

我安装的是8，看来8是没有资格编译14的。老老实实去下载openjdk9来编译。

hg clone http://hg.openjdk.java.net/jdk9/jdk9

openjdk9下载的文件特别少，里面有一个get_source.sh文件，通过它来下载全部剩下的文件。

#给这个文件开放权限
chmon 777 get_source.sh

#执行shell文件下载源代码
./ get_source.sh

下载所有代码之后再次运行bash configure 检查配置，发现还是报错，少了一个freetype
安装freetype

brew install freetype

安装完成之后竟然还是提示找不到freetype。上网找找看。
原来要自己指定freetype的安装路径

brew list freetype

这里要注意，每次编译的时候需要县make clean，清除上一次的结果再进行编译，不然容易出奇怪的问题。
编译错误1
出错。

clang: error: include path for libstdc++ headers not found; pass '-stdlib=libc++' on the command line to use the libc++ standard library instead [-Werror,-Wstdlibcxx-not-found]

试了很多方法，然后发现试因为xcode从10开始就删除了lstdc++，但是openjdk9的hotspot中用到了。所以自己从网上下载需要的库，然后copy到对应的文件夹。也可以安装老版本的xocde，然后从老版本copy到新版本。

https://github.com/imkiwa/xcode-missing-libstdc-

注意，copy的时候一定要注意安装目录是否正确。

在docker中搭建JAVA开发环境

其实步骤都写在了官方文档里面,但是完全照着做还是会有点问题.这里主要是把文档中没有明确写出来的东西记下来.

https://github.com/docker/labs/tree/master/developer-tools/java/

删除image

默认是无法直接删除container仍然有效的image的.想删除需要先停止/删除对应的container

# Stop one or more running containers
sudo docker container stop "container名"

# Remove one or more containers
sudo docker container rm "container名"
# Remove all stopped containers
sudo docker container prune

然后再删除image

# Remove one or more images
sudo docker image rm "image名"
# Remove unused images
sudo docker image prune

安装jboss

在文档中并没有直接给出jboss的安装过程,我们可以从github上找到jboss,clone下来之后build镜像.

mvn -f pom.xml package -Pdocker

打包war

文档中给出的javaee7的war的github地址是无效的,但是mvn的project还是在的.和jboss一样我们clone下来,自己用mvn package命令来打包war.

deploy

文档中用的是

FROM jboss/wildfly:latest
RUN curl -L https://github.com/javaee-samples/javaee7-simple-sample/releases/download/v1.10/javaee7-simple-sample-1.10.war -o /opt/jboss/wildfly/standalone/deployments/javaee-simple-sample.war

因为war包的地址失效,所以我替换成本地的war

FROM jboss/wildfly:latest
ADD target/javaee7-simple-sample.war /opt/jboss/wildfly/standalone/deployments/

生成镜像

docker image build -t javaee-sample .

运行

docker container run -d -p 8080:8080 javaee-sample

然后在浏览器中查看结果.

http://localhost:8080/javaee7-simple-sample/resources/persons

注意点

1.本地的文件拷贝不要用系统自带的cp,会有权限问题,用docker的copy或者add

2.拷贝文件的时候,一定要注意是否重命名.在浏览器中查看结果的时候,我就是因为少打了一个7导致始终得不到想要的结果.

3.遇事不决先重启!!遇事不决先重启!!遇事不决先重启!! windows我真的是受够了.

win10环境ubuntu子系统中安装docker

历经千辛万苦才成功,也是不容易.把各种坑记下来.

Win10的版本

虽然大家都是win10, 但是子版本不同,会造成docker是否可用.

docker是基于linux的,windows的支持确实不好.wsl2才完全支持, wsl1可以洗洗睡了.

WIN + R 打开运行栏,输入winver

可以确认win10的详细版本,只有2004以上才支持wsl2.

如果低于2004, 先升级到2004(windosw的update, 或者手动升级https://www.microsoft.com/en-us/software-download/windows10)

开启WSl2

用管理员权限打开powershell,输入以下命令开启wsl,并将wsl2设置为默认

dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
wsl --set-default-version 2

安装ubuntu

打开微软store, 从中下载ubuntu, 2020年7月这个时间点有3个版本,随便哪个都可以啦.

打开ubuntu

你可能觉得我有毛病,为什么打开ubuntu也要单独写个段落?

当然是为了提醒自己一些重要的事情.

一定要用管理员权限打开!!

一定要用管理员权限打开!!

一定要用管理员权限打开!!

第一次打开需要输入用户名和密码.

在ubuntu中安装docker

上面的步骤走完之后,就只需要按照官方文档来做就好了.

https://docs.docker.com/engine/install/ubuntu/

$ sudo apt-get update

$ sudo apt-get install \
    apt-transport-https \
    ca-certificates \
    curl \
    gnupg-agent \
    software-properties-common

$ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

$ sudo add-apt-repository \
       "deb [arch=amd64] https://download.docker.com/linux/ubuntu \
       $(lsb_release -cs) \
       stable"

$ sudo apt-get install docker-ce docker-ce-cli containerd.io

启动docker

如果出现了什么奇奇怪怪的错误,把杀毒软件关掉再试试!!!!!

$ sudo service docker start
$ sudo docker run hello-world

将当前用户加入docker group

因为docker的守护进程使用的端口是unix Socket 而不是tcp端口,而unix端口默认是root用户才可以访问, 所以之后每次docker的操作几乎都需要加上sudo,很麻烦.
所以,将当前登录的用户加入docker group之后,就可以省去sudo命令.

sudo groupadd docker             # 添加docker用户组
sudo gpasswd -a $USER docker     # 将登陆用户加入到docker用户组中
newgrp docker                    # 更新用户组
docker version                   # 确认不使用sudo时是否可以访问Docker daemon

搭建自己的博客(github+hexo)

为了加深记忆,也为了在以后想不起细节的时候可以查询,所以在这里记下步骤和坑.

本文大量参考了知乎文章　https://zhuanlan.zhihu.com/p/26625249

基于win10(2004 19041.330)

安装git

https://git-scm.com/downloads

从git官网下载安装包,安装完成之后,在cmd中输入 git -version 确认安装是否成功.

连接github

在cmd中输入 用户名和邮箱

git config --global user.name "GitHub 用户名"
git config --global user.email "GitHub 邮箱"

生成ssh key,名称和密码可以为空

ssh-keygen -t rsa -C "GitHub 邮箱"

添加ssh key到github.

在生成ssh key时,cmd回返回信息key生成在哪个文件夹,打开那个文件夹,复制里面的内容, 在github的 setting -> SSH and GPG keys -> SSH keys中新建SSH key,把前面复制的内容粘贴进来.

验证连接
在cmd中输入 ssh -T git@github.com

安装node

https://nodejs.org/en/
从node官网下载安装包,安装完成之后,在cmd中输入node -v 确认安装是否成功.

安装hexo

在任意路径创建文件夹(比如”MyBlog”),打开cmd,在MyBlog文件夹目录下,安装hexo

npm install -g hexo-cli

完成之后初始化blog(在MyBlog文件夹中创建blog文件夹,之后的博客文件都是在这个blog文件夹中)

hexo init blog

注意,上面两步操作中可能会出现一些乱七八糟的warn,无视就好.

完成初始化之后可以在本地测试一下.
在cmd中,进入blog目录(上面新建的文件夹),输入以下内容

hexo new test_my_site

hexo g

hexo s

打开浏览器, 输入localhost:4000,就可以看到例子了.

推送网站

在github上新建repository: XXXX.github.io
这里注意,repo的名字一定要用这种形式,不然无法访问

打开MyBlog->blog目录下的_config.yml文件,下拉到文件的最下方,修改为如下内容

deploy:
  type: git
  repo: git@github.com:YYYY/XXXX.github.io.git
  branch: master

YYYY : username
XXXX.github.io : 上面新建的repo名
注意,repo后面要加上.git

打开CMD,在MyBlog -> blog文件目录下,安装git插件

npm install hexo-deployer-git --save

然后输入

hexo clean
hexo d -g

完成之后,就可以在浏览器中输入 XXXX.github.io 来访问自己的blog了.

绑定域名

这次没做,所以先空着.以后申请了域名再来更新

更换主题

先空着

Markdown语法

比html更简单的标记语言.不过还是找个Markdown的编辑器来写更方便.我用的是Markdownpad http://www.markdownpad.com/
